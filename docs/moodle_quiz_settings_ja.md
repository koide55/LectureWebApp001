# Web アプリケーション脆弱性演習
## Moodle 小テスト設定リスト

この資料は、各回の講義スライドと演習に対応した Moodle 小テスト設定用の一覧である。  
Moodle 上で問題を作成するときに、どの問題形式を使うか、何を回答させるか、どう採点するかを整理する目的で使う。

## 基本方針

- ハンズオンは原則として小テスト化しない
- 小テスト化するのは「演習」と「宿題」
- 学生が答えやすいよう、作文問題では「番号付きで答える」指示を入れる
- 短い確認問題は選択式にしてもよい
- コード読解や比較は作文問題が向いている

## 共通設定の推奨

### 作文問題

- 問題形式: `作文`
- 解答欄:
  - `プレーンテキスト` または `HTMLエディタ`
- 回答文字数の目安:
  - 短文: 80-150字
  - 標準: 120-220字
  - 長文: 160-300字
- 問題文の末尾に必ず入れる指示:
  - `番号付きで答えてください`
  - `指定された項目をすべて含めてください`
- 手動採点を前提にする

### 選択式問題

- 問題形式: `多肢選択` または `組み合わせ`
- 用途:
  - 脆弱性分類
  - 対策と脆弱性の対応付け
  - 認証 / 認可 / 入力処理 / 出力処理 / 状態管理の分類

### ファイル提出を伴う課題

- 小テストより `課題` 活動が向いている
- ただし、説明文だけ小テストで回収してもよい

## 問題作成時の注意

- 問題文の最後に、必ず `番号付きで答えてください` を入れる
- 作文問題では、必須項目を箇条書きで示す
- 比較問題では、`safe` と `vulnerable` の両方に触れさせる
- 組み合わせ問題では、選択肢の数を増やしすぎない
- 第1回から第3回は、コード探索の観点を重視する
- 第4回以降は、脆弱性の本質と対策の対応を重視する
